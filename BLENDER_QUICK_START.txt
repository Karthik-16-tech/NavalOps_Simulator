For Web/Three.js Integration:
  → File → Export As → glTF 2.0 (.glb)
  → Settings: Include Materials, Include Normals
  → File size: ~15-30 MB (optimized)
  → Use in: Three.js, Babylon.js, A-Frame

For Game Engines:
  → File → Export As → Autodesk FBX (.fbx)
  → Settings: Selected Objects, Apply Modifiers
  → File size: ~50-100 MB
  → Use in: Unity, Unreal Engine, Godot

For Archival/Print:
  → File → Export As → Wavefront OBJ (.obj)
  → File size: ~200+ MB (high detail)
  → Use for: High-quality renders, printing

For Animation/VFX:
  → File → Export As → Alembic (.abc)
  → Preserves: Topology, deformations
  → Use in: Maya, Houdini, Cinema4D

❓ TROUBLESHOOTING:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Problem: "Script not running"
  → Switch to Scripting workspace
  → Verify script is fully opened
  → Click Play button or press Alt+P

Problem: Model is too small/far away
  → Press . (Numpad Period) to frame
  → Or press Home key
  → Or scroll mouse wheel to zoom

Problem: Model appears faceted/blocky
  → Add more subdivisions in script
  → Apply smooth shading (select → Right-click → Smooth)

Problem: Rendering is very slow
  → Reduce samples to 64
  → Use EEVEE instead of Cycles
  → Apply modifiers: Modifier → Apply

Problem: Can't see collections in Outliner
  → Right panel → Toggle "Collections" view
  → Check eye icons are visible

Problem: Need to start over
  → Ctrl+A → Select All
  → X → Delete All
  → Run script again (Alt+P)
